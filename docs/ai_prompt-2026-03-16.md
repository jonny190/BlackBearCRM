# AI Coding Assistant Prompt: CRM System Development

You are tasked with building a modern Customer Relationship Management system focused on relationship health and institutional memory. This is not a traditional transaction-focused CRM, but rather a "living file" system that treats accounts as evolving organisms.

## Core Technical Requirements

### 1. Account Health Scoring System
**Build a dynamic health scoring engine that:**
- Calculates real-time health scores combining:
  - Engagement frequency (meeting cadence, email responses)
  - Sentiment signals (extracted from communications)
  - Renewal proximity and risk indicators
  - Deal momentum and pipeline activity
- Stores historical health score data for trending analysis
- Provides configurable weighting for different health factors
- Supports threshold-based alerting when scores drop below defined levels

**Acceptance Criteria:**
- Health scores update within 15 minutes of new activity
- Scores are displayed as visual indicators (color-coded, trending arrows)
- Historical scoring data retained for minimum 2 years
- Configurable alert thresholds per account tier

### 2. AI-Powered Context Engine
**Implement an AI system that:**
- Generates 90-day account briefings for new account managers
- Creates pre-meeting summaries automatically
- Extracts and surfaces key decisions and commitments from communications
- Provides relationship gap analysis and recommendations

**Technical Implementation:**
- Integration with LLM API (OpenAI/Claude) for text analysis
- Document processing pipeline for emails, meeting notes, proposals
- Sentiment analysis on communication history
- Automated summary generation with key facts extraction

**Acceptance Criteria:**
- Briefings generated within 30 seconds of request
- Summaries include: recent activities, pending items, relationship status, next actions
- Key decisions automatically tagged and pinnable to account records

### 3. Relationship Mapping System
**Create a contact influence mapping feature that:**
- Visual network diagram of contacts at each client
- Tracks role, influence level, last interaction date
- Identifies relationship gaps (missing executive contacts, single points of failure)
- Flags accounts with only low-level contacts

**Technical Specifications:**
- Graph database or network visualization library
- Contact scoring algorithm based on title, interaction frequency, deal involvement
- Automated alerts for single-contact dependencies
- Export capability for relationship maps

**Acceptance Criteria:**
- Visual maps render within 3 seconds
- Support for 100+ contacts per account without performance degradation
- Automatic flagging when only contact is below director level

### 4. Activity Integration and Automation
**Build comprehensive activity capture:**
- Calendar integration (Google Calendar, Outlook) for automatic meeting logging
- Email integration for communication tracking
- Proposal and document tracking with follow-up automation
- CRM activity sync with minimal manual data entry

**Integration Requirements:**
- Google Workspace / Microsoft 365 API integration
- Email parsing and categorization
- Calendar event processing with account linking
- Automatic follow-up reminders based on activity types

**Acceptance Criteria:**
- 95% of meetings automatically logged within 1 hour
- Email threads linked to correct accounts with 90% accuracy
- Zero-manual-entry workflow for standard activities
- Follow-up prompts triggered based on configurable rules (e.g., 5 business days for proposals)

### 5. Dashboard and Reporting System
**Create role-based dashboards:**
- Account manager dashboard with health scores, action items, upcoming activities
- Team lead dashboard with team performance and at-risk account alerts
- Operations dashboard with data quality metrics and system adoption

**Dashboard Features:**
- Real-time health score visualization
- Activity timeline per account
- Configurable alert settings
- Export capabilities for reports
- Mobile-responsive design

**Acceptance Criteria:**
- Dashboard loads within 2 seconds
- Support for 500+ accounts without pagination issues
- Real-time updates via WebSocket connections
- Role-based access control

## Non-Functional Requirements

### Performance
- Page load times under 2 seconds for standard views
- Health score calculations complete within 15 minutes
- Support for 1000+ accounts and 10,000+ contacts
- 99.9% uptime SLA

### Data Quality and Compliance
- Automated data validation and cleansing
- Activity gap reporting (accounts with no activity >30 days)
- Audit trail for all data changes
- GDPR compliance for contact data handling

### User Experience
- Intuitive interface requiring minimal training
- Progressive disclosure of information
- Mobile-first responsive design
- Keyboard shortcuts for power users

## Technical Architecture Requirements

### Backend
- RESTful API architecture with clear endpoint structure
- Real-time notification system (WebSocket/Server-Sent Events)
- Background job processing for AI analysis and score calculations
- Comprehensive logging and monitoring

### Frontend
- Modern JavaScript framework (React/Vue/Angular)
- Component-based architecture
- State management for real-time updates
- Responsive CSS framework

### Database Design
- Relational database for core CRM data (accounts, contacts, activities)
- Time-series data structure for health score history
- Search optimization for contact and account lookups
- Data archival strategy for long-term storage

### Integration Layer
- OAuth 2.0 authentication for external services
- Webhook handling for real-time external updates
- Rate limiting and error handling for API integrations
- Data synchronization strategies for external systems

## Implementation Priority

**Phase 1 (MVP):**
1. Core account and contact management
2. Basic health scoring (engagement frequency only)
3. Manual activity logging with timeline view
4. Simple dashboard with health score visualization

**Phase 2 (AI Integration):**
1. Email and calendar integration
2. AI-powered briefing generation
3. Automated follow-up reminders
4. Advanced health scoring with sentiment analysis

**Phase 3 (Advanced Features):**
1. Relationship mapping visualization
2. Advanced reporting and analytics
3. Mobile application
4. Advanced AI recommendations

Build this system with clean, maintainable code, comprehensive error handling, and thorough testing. Focus on user experience and ensure the system genuinely reduces cognitive overhead for account managers while maintaining complete relationship context.
